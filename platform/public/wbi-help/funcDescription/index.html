# 功能说明

# 登录

平台用户均来自权限系统，可通过账号密码进行登录，不同用户角色拥有不同的操作权限。

# 角色权限

# 平台角色分类

# 1、管理角色

管理类角色，拥有平台所有操作权限，拥有所有项目下的操作权限。

# 2.运维角色

管理类角色，负责平台资源的管理、平台项目本身的监控查看。

# 3.业务角色

管理类角色，负责平台项目、项目分组的管理、拥有所有项目下线的操作权限。

# 4.资源角色

非管理类角色，可以开发及上传各类资源，如（主题、组件等），拥有自已上传资源的管理权限（不包括锁定功能），资源角色只可以删除自己上传的资源。

# 5.项目角色

非管理类角色，可以创建项目，拥有所创建项目的所有操作权限，特别说明不能查看其它项目。

# 项目角色分类

# 1.管理角色

拥有对所属项目的所有操作权限。

# 2.运维角色

拥有所属项目的配置、授权、发布权限。

# 3.开发角色

拥有所属项目的页面管理、服务资源管理、数据工作台（开发）权限。

# 4.资源角色

代表用户未授权任何角色，仅可以查看平台资源信息。

# 首页

# 应用对象：平台用户

用户登录成功后进入平台首页，首页左上角为平台LOGO及名称，中间为功能菜单，点击相关功能菜单即可进入对应的功能模块界面，最右边为用户信息，底部显示上次登录平台时间。

点击用户名称，下拉展开用户操作项：“用户信息”和“退出登录”。

点击“用户信息”可查看当前用户相关信息：用户名、用户角色、最近登录时间和最近登录IP。

点击“退出登录”退出平台。

# 项目管理

[功能描述]

项目管理，主要对个人项目、项目资源管理，以满足一站式开发服务。

[操作说明]

首页点击“项目管理”功能菜单进入项目管理功能界面。

项目管理列表展示方式有列表和卡片两种形式，点击右上角图标进行两种展示方式的切换。

# 1.1 项目分组管理

# 应用对象：平台角色(管理角色、业务角色)

项目分组管理包括分组的新增、修改、删除、查询。

分组管理由平台管理员控制,不开放给用户使用，用户需要新增、修改分组等操作需要向平台申请。

# 分组数据为树形结构

1.平台初始化时会设置一个默认的根节点分组

2.在需要添加分组的节点上添加分组，满足对多项目的分类管理

3.分组针对平台设置，全平台用户共用同一个分组树，由平台管理员控制分组信息的变更

4.由用户在某分组下创建项目并授权给其它用户，展示项目时可以切换展示只拥有项目的分组或所有分组

# 1.1.1 查询分组

[功能描述]

根据根节点信息查询下级节点，节点信息为空时获取根节点；可切换展示只拥有项目的分组信息。

# 1.1.2 新增分组

在需要添加分组的节点上点击鼠标右键出现更多操作选项卡，包括：新增分组、新增子分组、编辑分组、删除分组。

[操作说明]

点击“新增分组”，填写分组相关信息，新增分组需要填写分组名称（必填），分组描述（非必填）等相关信息。

# 1.1.3 编辑分组

[功能描述]

编辑分组即对已经存在在分组节点的分组信息作修改，编辑分组可对分组名称、分组描述、上级分组进行修改。

[操作说明]

点击“编辑分组”，对当前分组信息进行修改。

TIP

如果仅想改变分组的上级分组，可按住鼠标左键进行分组的拖拽。

# 1.1.4 删除分组

[功能描述]

删除分组是对项目分组的一个删除操作，删除分组的前提是在该分组不包括项目的情况下可删除该项目分组；如果该分组下已经包含了项目则可先将项目移动到其他分组再进行分组删除操作。

# 1.2 项目管理

# 1.2.1 项目管理

# 1.2.1.1 项目列表查看
# 应用对象：平台角色(管理角色、业务角色、项目角色),项目角色(管理角色)

[功能描述]

选择项目分组可查看分组下的项目列表信息。

在项目列表页面，用户可以对项目进行搜索①、新增②、③详情查看、上线/下线、编辑、删除等操作（根据权限不同可操作内容不同）。

# 1.2.1.2 新增项目
# 应用对象：平台角色(管理角色、业务角色、项目角色)

[功能描述]

新增项目：选择项目生产环境的运行模式，共享模式和独占模式，共享模式：使用平台托管服务器资源，一个服务器部署多个项目；独占模式：用户自有的服务器资源，由用户管理。需要平台连接服务器作初始化，初始化完成后可以发布服务；BS独占模式：通过系统浏览器默认访问，所有资源需要全新加载。

项目信息包括：项目名称、行业分类、行业公司、部门、专业分类、区域信息、版本、自定义标签、项目图标、负责人、联系方式、邮件地址等信息。

项目主题选择：选择系统主题、项目主题或者自定义主题。

[操作说明]

列表页左上角点击“新增”，进入项目信息填写页面，填写完成点击“创建”即可完成项目的新增。

# 1.2.1.3 编辑项目
# 应用对象：平台角色(管理角色、业务角色、项目角色),项目角色(管理角色、运维角色)

[功能描述]

项目编辑指的是对项目常用属性信息的编辑。

[操作说明]

编辑项目入口有两个：

入口一

列表处点击“编辑”进入项目基本信息编辑界面。

卡片式

入口二

项目详情页面点击右上角“编辑”进入项目基本信息编辑界面。

# 1.2.1.4 删除项目
# 应用对象：平台角色(管理角色、业务角色、项目角色),项目角色(管理角色)

[功能描述]

针对废弃项目作删除操作， 删除项目仅项目创建人员可删除（逻辑删除），项目状态必需为下线状态。

平台管理员权限可以进行物理删除，删除项目需要二次密码重鉴别。

[操作说明]

在项目列表找到需要删除的项目，点击操作项“删除”，输入登录密码校验成功即可删除该项目。

卡片式

# 1.2.1.5 项目详情

[功能描述]

点击项目列表中一个具体的项目，对项目的详细信息进行查看。

[操作说明]

在项目列表页，列表式：选择具体某一个项目点击操作项“查看”;卡片式列表:点击项目封面,进入项目详情页。

# 项目基本信息
# 应用对象：平台角色(管理角色、业务角色、项目角色),项目角色(管理角色)

项目信息包括：项目名称、行业分类、行业公司、部门、专业分类、区域信息、版本、自定义标签、项目图标、负责人、联系方式、邮件地址等信息。

# 数据资源管理
# 应用对象：平台角色（管理角色、业务角色、项目角色），项目角色（管理角色、运维角色） 。

[功能描述]

展示当前项目的数据库列表，应用对象可对每个数据源的开发、测试、生产三个环境数据进行查看。

# · 新增数据源

[功能描述]

通过对数据库类型（mysql、oracle、postgre）选择， 数据库IP，数据库端口，用户名，密码，配置属性、数据库名称等完成数据源添加，数据源分为开发环境和生产环境，在项目上线时会将开发环境数据库同步至生产环境。

[操作说明]

点击列表左上角按钮“新增数据源”进行新增数据源的相关信息填写，填写完成点击“确定”提交完成。

# · 修改数据源

[功能描述]

数据源配置信息修改，须在项目处理未上线状态才可以编辑。

[操作说明]

在列表处选择某一个数据源，点击操作项“编辑”进入数据源编辑页面，对数据源相关参数进行修改。

# · 删除数据源

[功能描述]

辑数据源配置信息删除，删除时需要密码重鉴别且须在项目未上线状态才可以删除。

[操作说明]

在列表处选择某一个数据源，点击操作项“删除”对数据源进行删除操作，删除数据源需要“二次密码校验”。

# · 数据源检测

[功能描述]

检测数源的网络连能性与连接测试。

[操作说明]

在列表处选择某一个数据源，点击操作项“查看”，进入数据源详情页面，可看到“数据源监测”入口。

# · 数据字典
# 应用对象：平台角色（管理角色、业务角色、项目角色），项目角色（管理角色、运维角色、开发角色）

[功能描述]

通过数据库连接查看选择生成数据字典，可浏览数据表字典信息，生成数据字典文档下载，使用screw组件功能。

[操作说明]

在列表处选择某一个数据源，点击操作项“查看”，进入数据源详情页面，可看到“数据字典”入口。

# 服务资源管理

提供服务资源展示、模型配置管理，分为：自动接口模型、平台自定接口模型、本地接口模型。

# · 自动化接口模型管理
# 应用对象：平台角色（管理角色、业务角色、项目角色），项目角色（管理角色、开发角色）

[功能描述]

查看项目下的接口模型列表信息。可新增接口模型，选择某一个接口模型可以查看接口模型详情，对接口模型进行重新发布。

# 新增接口模型

[功能描述]

为项目数据模型设定通用的接口模型，接口模型按用户需求发布。

用户选择项目下的数据源，通过数据源获取数据模型，勾选数据模型一次生成一个或多个接口模型。

接口服务的生成，接口模型定义后测试和发布是平台提供的服务。

在项目上线时将接口模型通过接口服务组件生成JAVA代码并编译打包到项目中,由具体项目提供接口服务。

[操作说明]

点击列表左上角按钮“新增接口模型”，选择对应数据源下的数据库列表进行接口模型的新增。

# 重新发布接口模型

[功能描述]

当数据表数量或结构变化时，需要重新生成相应接口模型。

[操作说明]

列表处选择某一个接口模型，点击操作项“重新发布”，重新选择数据库表的字段列表，选择完成后点击“确定”进行接口模型的重新发布。

# · 本地接口管理
# 应用对象：平台角色（管理角色、业务角色、项目角色），项目角色（管理角色、开发角色）

[功能描述]

本地接口需要遵从swagger规范，平台通过swagger文件导入接口模型。

# 新增接口模型

[功能描述]

平台通过swagger文件或地址导入接口模型。

[操作说明]

点击列表左上角按钮“新增接口模型”，填写所属系统名称、添加Swagger文件进行接口模型的新增。

# 1.2.2.1 项目上线
# 应用对象：平台角色（管理角色、项目角色），项目角色（管理角色、运维角色）

[功能描述] 将已完成开发的项目发布以项目运行时方式访问，发布可选择导航窗口模板设置相应的导航窗口。

发布的同时会将项目信息推送至资源库，可添加额外的标签信息，以备搜索，针对已推送至资源库的项目可作更新操作或发布为其它版本以作区分。

发布过程需要作二次密码重鉴别，发布的同时会向项目组成员自动推送发布邮件。

[操作说明]

列表处选择某一个项目，点击操作项“上线”，填写自定义标签、选择项目部署服务器、输入账户密码（校验），填写完成点击“发布项目”，等待项目部署。

项目上线包含三个流程：生成代码、构建项目、部署项目。上线等候过程中会在当前页显示部署日志。

# 1.2.3 项目页面管理

# 应用对象：平台角色(管理角色、业务角色、项目角色),项目角色(管理角色、开发角色)
# 1.2.3.1 页面列表

[功能描述]

用户创建的页面列表展示。

[操作说明]

在项目详情页面点击“项目页面管理”按钮，进入项目页面管理界面。

页面左侧为项目页面列表，点击右上角图标可切换列表和卡片两种展示方式。

用户可在此处对项目页面分组、项目页面进行：新增、修改、删除等操作;

页面分组操作：鼠标指向分组节点，分组操作项下拉展开，

可新增、修改组名称、添加子页面和删除分组（删除分组的前提是该分组下不包含有页面，如果包含页面，需要先把页面移动到其他分组再进行删除）操作。

新增分组 新增分组也可以点击左上角分组图标进行分组的新增。

页面操作：鼠标指向某一个页面节点，页面操作项下拉展开

可对页面进行修改名称、复制、删除操作（删除页面需进行二次密码校验）

# 1.2.3.2 页面新增

[功能描述]

用户对项目页面的新增设计。

[操作说明]

新增页面有两个入口：①页面左上角点击“十”号，②分组节点操作项“新增子页面”进行页面的新增（从页面分组处新增页面，页面所属分组自动获取）。

第二步 选择页面新增方式：新增空白页面和从布局模板中选择两种方式。

新增空白页面

新增空白页面进入工作台，操作台为空白，需要用户自主设计页面。

选择布局模板新增页面

选择布局模板新增页面，进入工作台，操作台内用户选择的模板页面已经展示出来，用户可以根据模板页面进行修改完成自己的页面设计。

第三步 在工作台中设计完成以后点击"保存"即可完成页面的新增。

# 1.2.3.3 页面内容编辑

[功能描述]

页面编辑包括页面基本信息的编辑和对页面内容设计的编辑。

[操作说明]

页面内容编辑

有相关编辑权限角色在查看页面时，页面右上角会显示一个“编辑”按钮。点击“编辑”按钮可对当前页面进行编辑。

编辑页面需跳转工作台，在编辑过程中可以点击“预览”按钮对修改效果进行预览，修改完成后点击“保存”即可完成页面编辑。

页面基本信息编辑

有相关编辑权限角色在查看页面基本信息时，页面右上角会显示一个“编辑”按钮。点击“编辑”按钮可对当前页面的基本信息进行编辑。页面基本信息编辑内容包括：页面名称、自定义标签和页面描述。

# 1.3 开发工作台

# 1.3.1 页面设计

# 应用对象：平台角色(管理角色、运维角色、项目角色),项目角色(管理角色、开发角色)

BI工作台即页面设计操作工作台。

# 1.3.1.1 工具栏

工具栏位于工作台顶部，右侧依次为①“切换主题”、②“辅助布局”、③“预览”和④“保存”四个操作按钮。

# · 预览

[功能描述]

对于处在编辑中的页面提供实时预览功能，并且能够提供数据实时更新，刷新即可获取最新数据，预览为本地预览，不会将当前最新数据存储至服务器。

# · 保存

[功能描述]

# · 辅助布局

[功能描述]

主要实现通过简单的SVG布局容器生成对应的页面配置文件，精简手工工作量。

相关规范：

type：组件类型分为布局容器、容器和组件。布局容器主要实现布局，下级组件可为布局容器、容器、组件。容器主要用于包裹组件。组件即为系统中的各个功能组件。组件只能放入容器中，而不能放入布局容器中。

lableValue：不同组件类型具有的属性可通过此属性进行区别，如背景、预设组件等。

column和row：列参数和行参数属性，可以为像素和相对值，可指定多个值，每个值用空格隔开，如：column: 200px 1fr，表示定义两列，第一列宽度为200px，另外一列宽度铺满剩余宽度。

columMerge和rowMerge：行合并和列合并参数，通过标线范围来确定合并范围，用 / 隔开，如：columMerge: 1 / 3，表示第一列和第二列合并。注意，合并参数不能重叠，比如1/3 2/4这种合并参数，合并参数也不能越界，如定义了n列的布局，那么标线不能大于n + 1，也不能小于1

[操作说明]

点击界面右上角辅助布局图标，上传设计好的svg文件，后台根据上传的svg文件生成布局代码，用户可以在辅助布局的基础上进行页面的设计。

[功能描述]

属性栏主要用于设置组件的初始化配置，主要配置项为：

# · 数据绑定

[功能描述]

组件的数据绑定操作，主要分为：

静态数据绑定：主要对组件数据做初始化处理或者静态组件的数据处理。

静态数据即为数据的默认值，即在组件渲染初期赋值。

变量绑定：即通过一套数据模型给组件绑定相关数据，变量可通过模型动态生成一套动态数据，或通过事件传递的数据。

选择事先预设好的变量，如果需要在渲染初期即渲染，则勾选默认请求选项，如果不勾选默认请求，则只绑定变量，并不渲染，渲染需要手动触发组件的$ refresh进行渲染，在手动渲染之前，都将会展示组件的默认静态数据。
