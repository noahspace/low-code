# 项目管理

# 新增项目

新增项目：选择项目生产环境的运行模式，共享模式和独占模式， 共享模式：使用平台托管服务器资源，一个服务器部署多个项目；

独占模式：用户自有的服务器资源，由用户管理。需要平台连接服务器作初始化，初始化完成后可以发布服务；BS独占模式：通过系统浏览器默认访问，所有资源需要全新加载。

项目信息包括：项目名称、行业分类、行业公司、部门、专业分类、区域信息、版本、自定义标签、项目图标、负责人、联系方式、邮件地址等信息。

【操作说明】

选择项目分组，在列表页左上角点击“新增”，进入项目信息填写页面，填写完成点击“创建”即可完成项目的新增。

# 工作台页面开发指南

# 工作台工具面板讲解

# · 工具栏

工具栏左侧为返回项目页面管理按钮和当前页面名称，右侧为预览和保存。

# · 大纲树

大纲树用于展示工作区所有组件，并展示组件直接的层级关系，可直接对某个组件进行激活、删除操作。

# · 组件库

组件库展示系统可使用的所有组件，采用二级分类设置，可点击某一分类进入组件列表展示，可长按左键拖动至工作区进行页面组件添加。

# · 图标库

图标库展示系统可使用的图标，采用二级分类设置，可点击某一分类进入图标列表展示，可长按左键拖动至工作区进行页面图标添加。

# · 图片库和插图库

图片库和插图库展示系统可使用的图片和插图，采用二级分类设置，可点击某一分类进入图片列表和插图列表展示，可长按左键拖动至工作区进行页面图片和插图添加。

# · 全局变量

全局变量用于设置当前页面的业务数据流中间件，可用于组件直接的事件数据传播和业务数据相关处理。

添加变量

在此输入全局变量名称，点击后面加号进行添加

变量配置

添加变量后即可进行变量配置，可配置项目有变量名称、变量描述、是否动态数据、处理函数

动态数据

动态数据主要区分此变量类型，如果此变量只是作为事件数据传播和业务数据中间件，那么则关闭动态数据，如果此变量作为业务数据请求，则为动态变量。

本地接口

本地接口为本地接口数据请求，为第三方开发接口的直接调用，可直接选择接口，并且支持绑定参数，参数可配置项为字段名称、默认值、绑定变量。

接口参数绑定变量只能绑定非动态变量，自动接口同样遵守。

自动接口

自动接口即平台生成的简单业务接口，包含对单表的增删改查。

自动接口使用需要先选择某张表，然后选择一个操作类型，再进行对操作类型下的配置进行配置。

1. 查询

具备分页配置、排序配置、和条件配置

2. 更新

具备条件配置、参数配置，参数配置即为要更新的字段数据。

3. 添加

只具备参数配置，参数配置即为要添加的字段数据。

4. 删除

只具备条件配置

处理函数

处理函数体为对数据输出进行处理的脚本，处理函数体必须要有 return 值，获取当前数据通过 data 获取，例如：

return data.name;

# · 工作区

工作区为组件编辑画板，可进行组件位置层级编辑，左键进行组件激活，激活的组件可进行相关配置，长按左键可进行拖动位置调整，右键可进行组件同级组件位置交换和组件删除。

# · 配置面板

配置面板为当前激活组件的配置面板，主要从组件的基本样式、数据和事件进行相关配置。

基础属性

基础属性为组件样式相关的一些配置。

数据绑定

每个组件都有不同的数据绑定项，在此配置项中，则此数据可进行动态配置。

数据默认值

数据默认值为数据的最初值，当没有绑定变量或变量值不存在的时候取默认值。

绑定变量

即绑定页面中的全局变量。

处理函数

处理函数体为对数据输出进行处理的脚本，处理函数体必须要有 return 值，获取当前数据通过 data 获取，例如：

return data.name;
1

是否初始化

开启初始化则在组件渲染时立即加载解析变量数据。如果取消那么组件只加载默认数据，此时绑定的变量数据并不会解析，变量数据需要人为通过事件触发数据解析。

受控组件数据绑定

受控组件数据绑定即当前数据需要同时更新多个组件数据或多个字段的时需要添加受控组件，系统默认绑定的受控组件为当前组件的当前字段，所以不需重复绑定。需要注意的是上面的数据处理配置只会对当前组件的当前字段数据影响，并不会影响受控组件的源数据。

# 事件交互

添加事件

添加事件可添加下方预设事件，也可输入自定义事件，事件不能重复添加。

事件配置

事件执行流程为：预设方法先后执行 -> 自定义方法，预设方法越先定义优先级越高。

预设方法

1. 绑定变量

绑定变量为将变量作为数据中间件，进行赋值操作，选择需要绑定的变量并且输入值或表达式，直接输入字符串则为具体的变量值，同时你也可以通过 ${} 写入表达式，表达式中可通过 args 获取参数列表，如果需要取事件第一个参数绑定变量则可通过 ${args[0]} 表达式绑定。

2. 加载数据

加载数据即手动触发数据绑定中的数据，只需要拷贝需要触发数据处理的组件 ID 即可。

# 工作台页面设计操作指南

项目页面设计工作开始前，先看一下需要设计页面的项目是否已存在，如果该项目未创建需要先创建项目。

进入工作台设计页面前需要先准备好设计页面所需的资源素材：

# · 资源素材切图

将设计好的UI效果图在设计源文件（psd/sketch/figma）中把所需的图片元素导出切图。（所需图标元素转换成字体图标或png切图）

# · 资源素材上传资源库

将切好的图片资源上传图片库（所需图标元素转换成字体图标后上传图标库）。

1、在首页点击功能模块“资源管理”，进入资源管理界面。

2、选择资源管理-“图片管理”，进入图片管理界面。

3、在左侧图片管理分类树，选择一个节点新增一个分组如：“页面素材”。

4、在新增分组之下可以创建一个子分组如“某某项目”可以按照项目或者自己的习惯来创建子分组。方便使用，也方便后期自己的素材资源管理。

5、在左侧分组树选择想要上传资源的分组，点击左上角“新增”，添加想要上传的图片素材。

# · 项目页面管理

项目页面管理：可对项目页面进行新增、编辑、删除等操作。

以下以新增空白页面为例来讲解工作台设计页面操作：

1、点击左侧功能菜单“项目管理”进入项目管理列表页，找到需要进行页面管理的项目。

2、进入项目详情页面点击“项目页面管理”按钮进入项目页面管理界面。

3、新增项目页面。

在左侧项目页面列表树选择一个节点，点击操作项“新增页面”，也可以选择对应的页面分组节点点击操作项“新增子页面”，填写页面基本信息：页面名称、描述。填写完成后点击“确定”按钮。

接下来选择页面模板：（可选择空白页和布局模板）新增空白页面。选择完成后点击“创建页面”跳转工作台。选择了新增空白页面，进入工作台后看到的空白，没有任何内容。

# 页面布局

在工作台内进行页面设计我们使用的是“组件库”-“布局”来对整个页面进行版块划分与排列。

着重说明：

使用组件布局来设计页面，首先需要UI设计师在前期设计页面UI效果图时比较规范，使用标准的栅格对页面进行布局设计。

以1920px1080px为例： 在UI设计前期，UI设计师将整个页面以24px为单元格将页面的行、列进行等分，整个页面的内容布局在网格框架内进行。如下图所示：

对UI设计效果图规范了解清楚后，根据设计规范在工作台内进行页面的布局设计。

1、在工作台中选择 “组件库”-“布局”，将整个页面的底层作为一个盒子容器放在工作区。

2、将底层容器进行列和行等分。

选中底层容器，容器属性设置：布局类型：网格布局；列、行设置中1fr代表的是1列/行，有多少行/列就需要输入多少个1fr,每个1fr用空格隔开。

3、搭建页面内容布局容器

点击左侧导航“组件库”图标，选择“布局”

打开组件“布局”，选择“盒子”用鼠标拖拽至右侧工作区域画布框内。当前我们看到的是一个24px*24px的小格子容器。

根据UI效果图的网格布局来设置当前容器的“项目属性”。

TIP

水平开始位置：指的是水平网格线第一根线的开始位置。水平结束位置：指的是水平网格线最后一根线的位置。

如下图，top导航纵向占了3个网格（3行），则它的水平开始位置为“1”，水平结束位置为“4”

TIP

垂直开始位置：指的是垂直网格线第一根线的开始位置。垂直结束位置：指的是垂直网格线最后一根线的位置。

如下图，top导航横向占了80个网格（80列），则它的垂直开始位置为“1”，垂直结束位置为“81”

top导航“项目属性”设置完成后效果如下：

# 内容填充

页面内容主要为组件、图标、图片、插图等元素组成。页面设计中组件库使用相对较多。

上传背景图片

根据页面设计效果，当前页面背景细节繁多，内容也比较复杂，所以在前期规划时将top以下整个页面的背景作为一个资源切图。

1、选中背景容器，将背景颜色改为透明，点击“确定”。

将容器背景颜色设为透明，主要用于填充透明背景或半透明背景的元素，如果不将容器背景设置为透明，白色底会显示出来，影响页面美观。如需填充图片或者不透明背景，则无需将容器背景设置为透明状态。

2、在“背景图片”处点击弹出资源选择框，找到之前已新增好的资源分组，在分组之下找到对应的图片资源，点击选择完成背景图片的添加。

TIP

在上传背景图片这里可以对背景大小、背景是否重复、背景定位、溢出内容是否显示进行设置。

页面内容设计

根据UI设计效果，在工作台内完成页面内容的设计，页面内容设计也是根据布局规范内来操作的。

以下图内容设计为例：

第一步：通过分析我们将该版块内容拆分为背景和内容两层，内容层包括标题①、标注②和内容展示区③三个部分。

第二步：在页面操作台选中版块容器，将容器内边距设置为20px(内边距是容器边框与内部内容的间距，设置内边距使内容不会贴着容器边框展示)，替换容器的边框背景图片。

第三步：根据前期UI网格规范，将此版块以行、列进行等分，完成内部布局排版。

标题行比较特殊，占用位置1行＜标题＜2行，这种情况我们可以给它自定义一个高度，如下图：

第四步：图片、文字内容填充。

填充图片型素材

选择需要填充素材的容器，背景颜色设置为透明，点击背景图片，在资源库中找到想应的素材，点击选中，完成图片的添加。可以对背景大小、背景是否重复、背景定位、溢出内容是否隐藏等样式属性进行设置。

填充文字内容

选择需要填充文字内容的容器，背景颜色设置为透明。

设置容器的容器属性：布局类型：网格布局，内容水平（内容在容器内水平对齐方式），内容垂直（内容在容器内垂直对齐方式）。

容器属性设置完成以后，在组件库中找到文字组件，将文字组件拖拽到容器内。

文字组件拖拽到容器内后显示的是默认的样式与文字内容。

我们可以在“基础属性”内修改文字内容的字体大小、文字颜色、字体加粗、行高和内容位置等属性。

如果需要修改文字内容，我们可以点击“数据绑定”，点击“文字内容”，对文字内容下的“数据默认值”进行修改，替换成自己想要的文字内容，修改完成后点击空白处即修改完成。

按照上述方法逐步完成整体页面的设计。设计完成点击“保存”完成页面设计，在设计途中也记得随时点击“保存”，对已完成设计内容进行保存，避免因为网络故障等因素导致的之前设计内容的丢失。在设计途中可以点击“预览”对已完成设计效果的查看，随时调整。

# 操作步骤

当用户需要添加图标库时，

第一步 在资源管理栏目下点开图标管理界面。

添加图标注意事项

当我们添加完成图标库资源后，在浏览查看时候会发现有些图标库内容串联了？（名字是当前图标库的，图标显示的是其他图标库资源的图标）

经过排查发现两个问题：

1、这一部分图标的code一样。

2、上传这个两个图标库都是从Iconfont库下载下来的图标，他们的库的命名是一样的，font-family: "iconfont"。

所以为了使图标不串，我们有以下两种方法来解决上述问题。

方法一：修改code

第一步：在Iconfont"我的项目"下找到需要修改code信息的图标，点击编辑图标，对当前图标的code进行修改，修改完成点击仅保存。

第二步：重新将图标下载到本地，将新下载下来的图标字体文件粘贴到fonts文件夹，替换原来的字体文件。

第三步：将新下载文件夹中的iconfont.css文件粘贴到原有文件夹中，替换font.css文件，命名为font.css，然后修改Css如下：

第四步：打开原文件夹中的data.json文件,对照font.css文件修改图标的code值，修改完成后将文件打包重新上传。图标就不会串联显示了。

方法二：修改库名称

第二种方法是相对简单一种的方法，不用来回查找修改以及下载图标。

第一步：打开串联显示的图标库文件夹，找到font.css文件，将css里面的库名进行修改。

第二步：找到data.json文件，将文件里面的每个图标的fclassName改成新修改的库名即可，修改完成后将文件打包重新上传。

# 新增插图库

# 操作步骤

当用户需要添加插图库时，

第一步 在资源管理栏目下点开插图管理界面。

第二步 选择插图库分类，在当前分类下的插图库列表处点击新增插图库按钮。

第三步 添加插图库zip包文件，添加完成后点击保存按钮即完成插图库的新增。

# 插图库zip包说明

用户在新增插图库前，需要先准备好需上传的插图库zip包文件。

zip包文件包含：illustration（插图库内容文件），data.json(插图库说明文件),screenshot（插图库封面图文件）。

zip包内的文件命名不能修改。

# 新增布局

# 新增组件库

# 操作步骤

第一步 在资源管理栏目下点开组件管理界面。

第二步 在组件资源管理栏目下点击新增组件库按钮。添加组件库zip包文件，添加完成后点击保存按钮即完成组件库的新增。

# 组件库zip包说明

用户在新增组件库前，需要先准备好需上传的组件库zip包文件。

zip包文件包含：①screenshot（单个组件封面文件夹），②data.json(组件库说明文件),③index.js(组件js文件)，④screenshot（组件库封面图文件）。

模板zip包内的文件命名不能修改。
